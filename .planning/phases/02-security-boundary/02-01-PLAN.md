  <action>
Create lib/proxy.sh following the established lib/*.sh module pattern (comment header, no direct execution, sourcing requirements documented).

Define these constants at the top:
```
CADDY_CONFIG_DIR="${XDG_CONFIG_HOME:-$HOME/.config}/rl"
CADDY_FILE="$CADDY_CONFIG_DIR/Caddyfile"
ANTHROPIC_PORT=9110
OPENAI_PORT=9111
```

Implement three functions:

**generate_caddyfile()** -- Creates `$CADDY_CONFIG_DIR` (mkdir -p) and writes `$CADDY_FILE` with this exact content (per D-01, D-02, D-03, D-04):
```caddyfile
{
    admin localhost:2020
}

http://127.0.0.1:9110 {
    reverse_proxy https://api.anthropic.com {
        header_up x-api-key "{env.ANTHROPIC_API_KEY}"
        header_up Host api.anthropic.com
    }
}

http://127.0.0.1:9111 {
    reverse_proxy https://api.openai.com {
        header_up Authorization "Bearer {env.OPENAI_API_KEY}"
        header_up Host api.openai.com
    }
}
```

IMPORTANT per research Pitfall 1: Anthropic uses `x-api-key` header, NOT `Authorization: Bearer`. Per Pitfall 4: admin API on non-default port 2020 to avoid conflicts with system Caddy. Per Pitfall 2: bind to 127.0.0.1 (SLIRP guest traffic appears as 127.0.0.1 on host, so this is both secure AND reachable). Per Pitfall 5: include `header_up Host` directives.

Use a heredoc with single quotes ('CADDYFILE') so the `{env.*}` placeholders are written literally, not expanded by bash.

**is_caddy_running()** -- Returns 0 if the Anthropic proxy port (9110) is responding. Use:
```bash
curl -sf -o /dev/null --connect-timeout 1 "http://127.0.0.1:$ANTHROPIC_PORT" 2>/dev/null
```
Per research: port probe is more reliable than pgrep or caddy status.

**ensure_caddy_running()** -- Idempotent guard that:
1. Calls `is_caddy_running` and returns 0 if already running
2. Checks `[ -z "${ANTHROPIC_API_KEY:-}" ]` and calls `die "ANTHROPIC_API_KEY not set. Export it in your shell before running rl."` if missing (per D-08: Anthropic key required, OpenAI optional)
3. Checks if `$CADDY_FILE` exists; if not, calls `generate_caddyfile`
4. Runs `caddy start --config "$CADDY_FILE" --adapter caddyfile 2>/dev/null` with error handling: `|| die "Failed to start Caddy proxy. Check 'caddy validate --config $CADDY_FILE'."`
5. After `caddy start`, wait briefly (sleep 1) then verify with `is_caddy_running || die "Caddy started but proxy not responding on port $ANTHROPIC_PORT."` to catch silent startup failures

Do NOT validate or require OPENAI_API_KEY (per D-08, it is optional). For the Caddyfile, if OPENAI_API_KEY is unset, Caddy may insert empty string or fail -- this is an open question from research. For now, do not add a fallback; the user just won't use Codex if they don't have the key. Note this as a comment in the code.

All code must pass `shellcheck --severity=warning`. Use `local` for function-scoped variables.
