**lib/ssh.sh** -- SSH and tmux session management:

1. No shebang (sourced file). Comment header: `# ssh.sh -- SSH connection and tmux session management`

2. `wait_for_ssh()` function (per Pitfall 1 from RESEARCH.md):
   - Takes vm_name and optional timeout (default 60 seconds).
   - Phase 1: Wait for `$AQ_STATE_DIR/$vm_name/ssh-port.conf` file to appear. Poll every 1 second.
   - Phase 2: Read port from file. Poll SSH connectivity: `ssh -o ConnectTimeout=2 -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -p "$port" root@localhost true 2>/dev/null`. Retry every 1 second.
   - If total elapsed exceeds timeout: return 1.
   - On success: return 0.

3. `cmd_code()` function (per SESS-01, D-04, D-05, D-06, D-12):
   - Read vm_name via `get_saved_vm_name` or die "No airlock for this repo. Run 'rl new' first."
   - Check if aq VM directory exists at `$AQ_STATE_DIR/$vm_name`. If not: die "VM '$vm_name' not found. Run 'rl new' to create."
   - If VM is not running (`! is_vm_running "$vm_name"`):
     - Auto-start it (per Open Question 3 recommendation from RESEARCH.md): `info "Starting VM '$vm_name'..."`, run `aq start "$vm_name"`, then `wait_for_ssh "$vm_name" 60 || die "VM started but SSH not available. Run 'rl status' to check."`.
   - Read SSH port: `ssh_port=$(get_ssh_port "$vm_name") || die "VM '$vm_name' is not running. Run 'rl status' to check."`
   - Connect with tmux attach-or-create (per Pattern 4 from RESEARCH.md):
     ```
     ssh -o ConnectTimeout=5 \
         -o StrictHostKeyChecking=no \
         -o UserKnownHostsFile=/dev/null \
         -p "$ssh_port" \
         root@localhost \
         -t "cd /root/repo 2>/dev/null; tmux new-session -A -s rl"
     ```
   - If SSH fails: die "SSH connection failed. Run 'rl status' to check VM state." (per D-12: fail immediately, no auto-retry)

**lib/vm.sh** -- add `cmd_new()` function:

Add to the EXISTING lib/vm.sh file (which already has `is_vm_running`, `cmd_status`, `cmd_rm`).

`cmd_new()` function (per VM-01, D-01, D-02, D-03, D-07):

1. Get VM name: `vm_name=$(get_vm_name)` (basename of cwd, per D-01)
2. Check if already exists: if `.rl/vm-name` file exists AND the saved name matches, die "VM already exists. Use 'rl code' to connect or 'rl rm' to destroy." (per D-03). Also handle cross-repo collision: if `$AQ_STATE_DIR/$vm_name` exists but `.rl/vm-name` doesn't, die "A VM named '$vm_name' already exists (from another repo). Rename this directory to avoid collision." (per Pitfall 2)
3. Create VM with spinner:
   - `spinner_start "Creating VM"`
   - `aq new "$vm_name" 2>/dev/null` -- aq outputs name to stdout, "Created:" to stderr
   - If aq new fails: `spinner_stop "Failed"` then die with error.
   - `spinner_stop "VM created"`
4. Start VM with spinner:
   - `spinner_start "Booting VM"`
   - `aq start "$vm_name" 2>/dev/null`
   - If aq start fails: `spinner_stop "Failed"` then die.
   - `spinner_stop "VM booted"`
5. Wait for SSH:
   - `spinner_start "Waiting for SSH"`
   - `wait_for_ssh "$vm_name" 60 || { spinner_stop "Failed"; die "SSH not available after 60s. VM may have failed to boot."; }`
   - `spinner_stop "SSH ready"`
6. Provision guest with spinner (per VM-01):
   - `spinner_start "Installing packages"`
   - Run provisioning script via aq exec:
     ```
     aq exec "$vm_name" <<'PROVISION'
     set -e
     apk add --no-cache tmux git
     mkdir -p /root/repo
     echo "PROVISION_OK"
     PROVISION
     ```
   - Capture output. Check that it contains `PROVISION_OK` (per Pitfall 7: verify provisioning succeeded).
   - If provisioning fails or sentinel missing: `spinner_stop "Failed"` then die "Guest provisioning failed. Try 'rl rm' and 'rl new' again."
   - `spinner_stop "Packages installed"`
7. Save state:
   - `save_vm_name "$vm_name"` (writes to `.rl/vm-name`, creates `.rl/`, updates .gitignore per D-02)
8. Final output:
   - `success "Airlock '$vm_name' ready"`
   - `info "Run 'rl code' to connect"`
