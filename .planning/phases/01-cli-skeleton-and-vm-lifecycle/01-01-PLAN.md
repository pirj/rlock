Create `lib/` directory and two shared modules. Both files must be Bash 3.2 compatible (no associative arrays, no `${var,,}`, no `readarray`, no `|&`).

**lib/util.sh** -- shared utilities:

1. `die()` function: prints error message to stderr in red (using $RED/$RESET from ui.sh), exits 1. Format: `printf '%sError:%s %s\n' "$RED" "$RESET" "$msg" >&2`

2. `check_dependency()` function: takes `cmd` and `install_hint` args. Uses `command -v "$cmd" >/dev/null 2>&1`. On failure calls `die "$cmd not found. Install: $hint"` (per D-11).

3. `check_all_deps()` function: checks for `aq` (hint: `brew install pirj/tap/aq`), `qemu-system-aarch64` (hint: `brew install qemu`), `git` (hint: `brew install git`), `ssh` (hint: `Install OpenSSH`), `tmux` (hint: `brew install tmux`). Note: do NOT check for `caddy` -- that is Phase 2.

4. `.rl/` state management functions:
   - `RL_DIR=".rl"` constant
   - `AQ_STATE_DIR="$HOME/.local/share/aq"` constant
   - `get_vm_name()`: outputs `basename "$(pwd)"` -- the repo directory name (per D-01)
   - `get_saved_vm_name()`: reads `.rl/vm-name` file, returns 1 if not found. Usage: `vm_name=$(get_saved_vm_name) || die "No airlock for this repo. Run 'rl new' first."`
   - `save_vm_name()`: takes vm_name arg, calls `ensure_rl_dir`, writes to `.rl/vm-name` via `printf '%s' "$1" > "$RL_DIR/vm-name"`
   - `ensure_rl_dir()`: `mkdir -p "$RL_DIR"`, then adds `.rl/` to `.gitignore` if not already present (per D-02). Check with `grep -qxF '.rl/' .gitignore`. If `.gitignore` does not exist, create it with `.rl/` as content.

5. `get_ssh_port()` function: takes vm_name, reads `$AQ_STATE_DIR/$vm_name/ssh-port.conf`. Returns 1 if file doesn't exist. NEVER cache the port -- always read fresh from aq state (per Pitfall 3 in RESEARCH.md).

**lib/ui.sh** -- terminal UX:

1. `setup_colors()` function: auto-detect terminal color support (per D-09). Check `[ -t 1 ]` (stdout is TTY) AND `[ -n "${TERM:-}" ]` AND `tput colors >= 8`. Set variables: `RED=$(tput setaf 1)`, `GREEN=$(tput setaf 2)`, `YELLOW=$(tput setaf 3)`, `BLUE=$(tput setaf 4)`, `BOLD=$(tput bold)`, `RESET=$(tput sgr0)`. When conditions fail, set all to empty string.

2. Output helper functions:
   - `info()`: `printf '%s%s%s\n' "$BLUE" "$1" "$RESET"`
   - `success()`: `printf '  %s✓%s %s\n' "$GREEN" "$RESET" "$1"`
   - `warn()`: `printf '%s%s%s\n' "$YELLOW" "$1" "$RESET" >&2`

3. Braille spinner (per D-07, D-08):
   - `SPINNER_CHARS` indexed array: `('⠋' '⠙' '⠹' '⠸' '⠼' '⠴' '⠦' '⠧' '⠇' '⠏')` -- a smooth clockwise dot-spinner sequence
   - `SPINNER_PID=""` tracking variable
   - `spinner_start()`: takes message arg. Launches background loop that `printf '\r  %s %s' "${SPINNER_CHARS[$((i % ${#SPINNER_CHARS[@]}))]}" "$msg"` to stderr every 0.08 seconds, incrementing i. Gate on `[ -t 2 ]` -- if stderr is not a TTY, print a single line `printf '  %s\n' "$msg" >&2` instead of spinning (per Pitfall 4). Store background PID in `SPINNER_PID`.
   - `spinner_stop()`: `kill "$SPINNER_PID" 2>/dev/null; wait "$SPINNER_PID" 2>/dev/null; SPINNER_PID=""`. Clear the spinner line and print the final message: `printf '\r  %s✓%s %s\n' "$GREEN" "$RESET" "$1" >&2`. When not a TTY, just `printf '  %s\n' "$1" >&2`.

4. Call `setup_colors` at source time (when the file is sourced, colors get initialized immediately).

Both files must NOT have a shebang (they are sourced, not executed directly). Both must start with a comment header explaining their purpose.

Create two files. Both Bash 3.2 compatible.

**rl** -- main CLI entry point (executable):

1. Shebang: `#!/bin/bash` (NOT `#!/usr/bin/env bash` -- system bash on macOS is at `/bin/bash`)
2. `set -euo pipefail`
3. Resolve script directory: `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` then `LIB_DIR="$SCRIPT_DIR/lib"`
4. Source shared modules (always needed): `. "$LIB_DIR/util.sh"` then `. "$LIB_DIR/ui.sh"`
5. Check all dependencies: `check_all_deps`
6. Parse subcommand: `cmd="${1:-help}"` then `shift 2>/dev/null || true`
7. Case-statement dispatch (per Pattern 1 from RESEARCH.md):
   - `new)` -- source vm.sh and ssh.sh (ssh.sh will be created in Plan 02; for now add a comment `# . "$LIB_DIR/ssh.sh"` as placeholder), call `cmd_new "$@"`. Since cmd_new doesn't exist yet, guard with: `die "Command 'new' not yet implemented."`
   - `code)` -- similar placeholder: `die "Command 'code' not yet implemented."`
   - `status)` -- source vm.sh, call `cmd_status "$@"`
   - `rm)` -- source vm.sh, call `cmd_rm "$@"`
   - `help|--help|-h)` -- call `cmd_help`
   - `*)` -- `die "Unknown command '$cmd'. Run 'rl help' for usage."`

8. `cmd_help()` function defined before the case statement:
```
usage: rl <command>

commands:
  new      Create a new isolated VM for the current repo
  code     Connect to the VM's coding session
  status   Show the current repo's airlock status
  rm       Destroy the VM and clean up resources

Run 'rl <command> --help' for command-specific help.
```
Print using `printf` or `cat <<EOF`, NOT `echo -e`.

9. Make file executable: (executor should run `chmod +x rl`)

**lib/vm.sh** -- VM lifecycle commands:

1. No shebang (sourced file). Comment header: `# vm.sh -- VM lifecycle commands (wraps pirj/aq)`

2. `is_vm_running()` function: takes vm_name. Checks `$AQ_STATE_DIR/$vm_name/process.pid` exists AND `kill -0 "$(cat "$pid_file")" 2>/dev/null` succeeds. Returns 0 if running, 1 if not.

3. `cmd_status()` function (per D-10, VM-03):
   - Read vm_name via `get_saved_vm_name` or die with "No airlock for this repo. Run 'rl new' first."
   - Check if aq VM directory exists at `$AQ_STATE_DIR/$vm_name`. If not: print `"$vm_name: ${RED}not found${RESET} (VM may have been removed externally)"` and return 1.
   - If running (via `is_vm_running`): read PID from `$AQ_STATE_DIR/$vm_name/process.pid`, read SSH port from `get_ssh_port` if available. Print: `"$vm_name: ${GREEN}running${RESET} (pid $pid, ssh:$port)"` or `"$vm_name: ${GREEN}running${RESET} (pid $pid)"` if no port.
   - If not running: print `"$vm_name: ${YELLOW}stopped${RESET}"`.
   - Output format is a single line (grep-friendly per specifics in CONTEXT.md).

4. `cmd_rm()` function (per VM-02):
   - Read vm_name via `get_saved_vm_name` or die with "No airlock for this repo. Run 'rl new' first."
   - Check if aq VM directory exists. If it does:
     - Run `aq rm "$vm_name"` to stop VM and remove aq state (per Pattern 3: aq rm handles stop + directory removal).
     - If `aq rm` fails (e.g., VM already gone), warn but continue cleanup.
   - If aq VM directory does NOT exist: print warning `"VM '$vm_name' not found in aq -- may have been removed externally"` (per Pitfall 5).
   - Always remove `.rl/` directory: `rm -rf "$RL_DIR"` -- clean up local state regardless.
   - Print success message: `success "Airlock '$vm_name' destroyed"`

After creating both files, run `chmod +x rl`.
